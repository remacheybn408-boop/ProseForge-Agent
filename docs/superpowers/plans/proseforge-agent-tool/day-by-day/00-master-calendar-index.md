# ProseForge Agent 90-Day Execution Calendar

Start date: 2026-06-26.

This calendar turns the large architecture plan into daily work. Each day has a concrete output, verification action, acceptance gate, and stop condition. The plan assumes one primary engineer plus optional reviewer time. If a day slips, keep the day order and move the dates forward; do not skip acceptance gates.

## Daily Workbook Contract

Every daily workbook generated by the future tool must contain:

- Date and day number.
- Phase name.
- Primary objective.
- Context to read.
- Morning build block.
- Midday verification block.
- Afternoon integration block.
- Closeout block.
- Files expected to change.
- Commands expected to run.
- Human review questions.
- Acceptance checklist.
- Memory update instructions.
- Risk note for the next day.

## Non-Negotiable Gates

- No provider is considered adapted until it has config validation, fake transport tests, error mapping tests, capability metadata, and a certification record.
- No memory feature is considered complete until retrieval output includes source references, confidence, reason for inclusion, and reason for exclusion.
- No writing workflow is considered complete until chapter output updates project state, memory, reports, and daily workbook recommendations.
- No release is considered complete until the demo works with the fake provider and at least one real provider profile in the local environment.
